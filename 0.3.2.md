## v0.3.2

- **新增**:
    - **持久化变量系统**: 引入了全新的变量持久化方案，数据将跨蓝图执行、跨服务器重启自动保存。
    - **全局持久化变量**: 
        - 新增 `获取全局变量 (持久化)` 节点。
        - 新增 `设置全局变量 (持久化)` 节点。
        - 数据统一存储在世界存档目录下的 `mgmc_variables.json` 中。
    - **实体持久化变量**:
        - 新增 `获取实体变量 (持久化)` 节点，支持通过实体对象或 UUID 绑定数据。
        - 新增 `设置实体变量 (持久化)` 节点，实现针对特定实体的持久化状态跟踪（如等级、属性等）。
        - 数据统一存储在世界存档目录下的 `mgmc_entity_variables.json` 中。

- **调整**:
    - **官网域名变更**: 将模组内所有官网链接及文档链接后缀由 `opens.ltd` 统一变更为 `nb6.ltd`。

- **修复**:
    - **蓝图失效问题**: 修复了在退出并重新进入世界后，原本的或新创建的蓝图会失效的 Bug。该问题是由于服务器停止时过度清理了全局事件节点定义导致的。

## v0.3.2 (English)

- **New Features**:
    - **Persistent Variable System**: Introduced a brand new variable persistence solution. Data will be automatically saved across blueprint executions and server restarts.
    - **Global Persistent Variables**: 
        - Added `Get Global Variable (Persistent)` node.
        - Added `Set Global Variable (Persistent)` node.
        - Data is stored in `mgmc_variables.json` within the world save directory.
    - **Entity Persistent Variables**:
        - Added `Get Entity Variable (Persistent)` node, supporting binding data via entity objects or UUIDs.
        - Added `Set Entity Variable (Persistent)` node for persistent state tracking for specific entities (e.g., level, attributes).
        - Data is stored in `mgmc_entity_variables.json` within the world save directory.

- **Adjustments**:
    - **Official Website Domain Change**: Updated all official website and documentation links from `opens.ltd` to `nb6.ltd`.

- **Bug Fixes**:
    - **Blueprint Failure Issue**: Fixed a bug where existing or newly created blueprints would fail after exiting and re-entering a world. This was caused by excessive clearing of global event node definitions when the server stopped.
