# /mgrun 命令

`/mgrun` 是 Maingraph for MC 的核心指令，用于在游戏内手动触发特定的蓝图逻辑。

## 命令语法

```mcfunction
/mgrun <蓝图名称> <事件名称> [参数...]
```

### 参数说明

- **`<蓝图名称>`**: 
  - 必填。
  - 蓝图名，即为在 蓝图管理器 中注册的蓝图名称。
  - 例如：输入 `test` 将调用 `test` 蓝图。

- **`<事件名称>`**: 
  - 必填。
  - 传递给蓝图内 [[当运行【斜杠】mgrun命令时]] 节点的“事件名称”端口。
  - 你可以根据此参数在蓝图内部实现不同的分支逻辑。

- **`[参数...]`**: 
  - 可选。
  - 传递给蓝图的自定义参数列表，多个参数之间用 **空格** 分隔。
  - 蓝图内部将通过“参数”端口（LIST 类型）接收这些值。

## 使用示例

### 1. 基础触发
触发 `hello 蓝图，并传递事件名 `start`：
```mcfunction
/mgrun hello start
```

### 2. 携带参数
触发 `teleport` 蓝图，传递事件名 `to` 和坐标参数：
```mcfunction
/mgrun teleport to 100 64 -200
```
*蓝图内部将收到参数列表：`100|64|-200`*

## 权限说明
- 默认情况下，所有玩家都可以执行此命令（`requires(s -> true)`）。
- *注意：具体蓝图逻辑是否涉及敏感操作（如修改世界、给予物品），取决于蓝图自身的节点设计。*

## 常见问题
- **找不到蓝图**：请检查蓝图管理器是否有对应蓝图，或查看 `saves/<你的存档>/mgmc_blueprints/` 目录。
